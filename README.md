# Mythos Log

Mythos Log is a native SwiftUI self-improvement app that treats real habits like stat training. Each stat has a baseline, surplus effort becomes charges, and weekly review is the moment where progress, stagnation, decay, and level-ups resolve.

## What is in V1

- SwiftUI app target for iPhone/iPad-ready layouts
- SwiftData persistence with local-only storage
- Dashboard, Character Sheet, Habits, Weekly Review, History, Settings, and Onboarding
- Weekly progression engine with charges, one-step level-ups, stagnation, and regression
- WidgetKit extension with small, medium, and accessory rectangular widgets
- App Intents / Shortcuts for core logging and navigation actions
- Local notification scheduling for daily, evening, and weekly review reminders
- Sample data seeding, export/import, and debug tools
- Unit tests covering progression, tiers, descriptors, streaks, and week math

## Core Progression Rules

The main weekly resolution logic lives in [`MythosLog/Domain/ProgressionEngine.swift`](./MythosLog/Domain/ProgressionEngine.swift).

- Each stat has a current baseline.
- At weekly review, actual value is compared to the baseline.
- `excess = max(0, actual - baseline)`
- `chargesEarned = excess`
- Stored charges carry forward.
- If the user both met baseline and has at least `4` stored charges, one level-up happens:
  - spend 4 charges
  - increase level by 1
  - increase baseline by 1
- Decay is configurable:
  - below baseline for multiple resolved weeks triggers stagnation
  - two zero weeks can remove a stored charge
  - prolonged underperformance can reduce baseline and level

Editable defaults for tiers, stat descriptors, colors, and seeded habits live in [`MythosLog/Domain/TrainingArcConfig.swift`](./MythosLog/Domain/TrainingArcConfig.swift).

## Widgets and Shortcuts

- Widgets read a shared JSON snapshot generated by the app through the app group in [`MythosLog/Support/WidgetSnapshot.swift`](./MythosLog/Support/WidgetSnapshot.swift).
- App Shortcuts cover gym, journal, reading, meditation, curiosity, generic habit completion, dashboard open, and weekly review open.
- External logging is prepared through a small ingestion layer in [`MythosLog/Domain/ExternalEventService.swift`](./MythosLog/Domain/ExternalEventService.swift) and URL parsing in [`MythosLog/Support/DeepLink.swift`](./MythosLog/Support/DeepLink.swift).

## Running the App

1. Open [`MythosLog.xcodeproj`](./MythosLog.xcodeproj) in Xcode.
2. Select the `MythosLog` scheme.
3. Run on an iPhone simulator.

The app is configured for an app group so the widget and app can share local state. For simulator builds this should work as-is. For running on a physical device, you may need to choose your Apple Development team in Signing and let Xcode provision the app-group entitlement.

## Running Tests

From Xcode:

- Run the `MythosLog` test action.

From Terminal:

```sh
DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer \
xcodebuild -scheme MythosLog \
  -destination 'platform=iOS Simulator,name=iPhone 17,OS=26.3.1' \
  CODE_SIGNING_ALLOWED=NO test
```

## Notes

- The app is intentionally local-first and does not require a backend.
- Cloud sync is not enabled in V1, but the model layer is separated cleanly enough for later SwiftData + CloudKit work.
- The project file was rebuilt to include the app, widget, and test targets explicitly. If it ever needs regeneration, the Ruby script in `Scripts/` documents that setup.
